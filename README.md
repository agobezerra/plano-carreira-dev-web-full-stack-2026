## Introdução

Este documento apresenta um plano de carreira estruturado para orientar a evolução profissional de Antonielle Gomes Bezerra rumo à área de atuação de Desenvolvimento Web / Full Stack. O conteúdo foi elaborado com base em seu perfil, experiência prévia, interesses técnicos e disponibilidade semanal de estudos.

O objetivo é fornecer uma visão clara e objetiva do caminho a seguir, destacando atividades do cotidiano, competências essenciais, roadmap de aprendizado e orientações para construção de portfólio e preparação de entrevistas.

## Experiência prévia e perspectivas futuras

Por que a carreira de desenvolvedor Web / Full-Stack combina com Antonielle Gomes Bezerra:

- Antonielle gosta de resolver problemas e prefere trabalhar com código, o que é central no desenvolvimento.
- Já está cursando Engenharia de Software, o que reduz muito o tempo até o nível júnior.
- Seu interesse declarado em desenvolvimento web encaixa perfeitamente.
- Sua experiência como professor e pesquisador em áreas exatas lhe dá forte base lógica e didática — isso acelera o aprendizado e o torna ótimo em documentar, explicar e estruturar soluções.

### Vantagens

- Mercado amplo e com alta demanda.
- Possibilidade de atuar em front-end, back-end ou full stack.
- Evolução rápida com prática constante.

### Desafios

- Ritmo acelerado de mudanças nas tecnologias.
- Necessidade de construir portfólio sólido.

### Mercado

Muito aquecido, especialmente para quem domina frameworks modernos. A demanda varia por região e experiência, mas é uma das portas de entrada mais acessíveis para transição.

### Visão Geral da Carreira

O Desenvolvedor Web / Full-Stack é responsável por criar aplicações completas, atuando tanto no front-end quanto no back-end. No dia a dia, esse profissional desenvolve interfaces interativas, implementa regras de negócio, integra bancos de dados e garante o funcionamento eficiente de sistemas web.

Além disso, participa de revisões de código, resolução de problemas técnicos e colaboração com equipes multidisciplinares. As principais atividades incluem:

- Construção de interfaces utilizando HTML, CSS e JavaScript.
- Desenvolvimento de APIs e serviços no back-end.
- Integração com bancos de dados relacionais e não relacionais.
- Otimização de performance e correção de bugs.
- Uso de ferramentas de versionamento e colaboração.

### Competências Essenciais

Para atuar como Desenvolvedor Web / Full-Stack, é necessário desenvolver um conjunto de habilidades técnicas e comportamentais. Entre as competências centrais estão o domínio de lógica de programação e JavaScript, conhecimento sólido em HTML5 e CSS3, além da capacidade de desenvolver APIs utilizando Node.js e Express.

Também é fundamental compreender o funcionamento de bancos de dados SQL e NoSQL. Competências complementares incluem o uso de TypeScript para maior robustez no código e noções de DevOps, como automação e containers.

Entre as ferramentas e tecnologias relevantes estão Git e GitHub para versionamento, React para construção de interfaces e Node.js para desenvolvimento back-end.

## Roadmap de 180 Dias

O plano de estudos foi adaptado para uma carga de 8 horas semanais, distribuídas ao longo de três meses.

### Mês 1 e Mês 2 — Fundamentos

- **Semanas 1 a 4:** Revisar lógica de programação com foco em JavaScript, criando scripts simples utilizando estruturas de repetição, funções e manipulação de arrays e objetos.
- **Semanas 5 a 8:** Estudo de HTML5 e CSS3, com desenvolvimento de duas páginas estáticas responsivas para consolidar os conceitos.

### Mês 3 e Mês 4 — Prática

- **Semanas 9 a 12:** Estudo de React, explorando componentes, propriedades e gerenciamento de estado, culminando na criação de uma aplicação simples, como uma lista de tarefas.
- **Semanas 13 a 16:** Estudo de Node.js e Express, com criação de rotas, middlewares e APIs REST, além da implementação de uma API conectada a um banco de dados.

### Mês 5 e Mês 6 — Portfólio e Preparação

- **Semanas 17 a 20:** Integração do front-end em React com o back-end em Node.js, incluindo autenticação básica.
- **Semanas 21 a 24:** Finalização do projeto de portfólio, documentação no GitHub e preparação de materiais profissionais.

## Projeto de Portfólio

O projeto sugerido é uma **Plataforma de Estudos Personalizados**, que permite ao usuário criar uma conta, registrar metas de estudo e acompanhar seu progresso.

**Escopo:**

- Interface em React com páginas de login, dashboard e gerenciamento de metas.
- API em Node.js com operações CRUD.
- Banco de dados em MongoDB ou PostgreSQL.

**Critérios de aceitação:**

- Autenticação funcional.
- CRUD completo.
- Interface responsiva.

Recomenda-se utilizar a experiência prévia como professor para criar uma experiência de usuário clara e bem documentada.

## Preparação para Entrevistas

A preparação envolve domínio técnico e boa comunicação. Entre as perguntas comuns:

- Apresentação pessoal, destacando formação e motivação.
- Descrição de projetos desenvolvidos, com foco em decisões técnicas.
- Estratégias para resolução de bugs.
- Explicação do ciclo de vida de componentes React.
- Definição de APIs REST.

A trilha sugerida é a **Formação Full-Stack Developer da DIO**, que oferece conteúdos alinhados ao roadmap e projetos guiados que fortalecem o portfólio. O acesso é gratuito e pode ser integrado ao plano de estudos apresentado.
